# SQL 쿼리 튜닝 도구 개발 계획서

## 📋 프로젝트 개요
- **프로젝트명**: SQL Query Tuning Assistant
- **목적**: 사내 시스템의 SQL 쿼리를 AI를 활용해 자동으로 튜닝하고 비교 분석하는 도구 개발
- **개발 방식**: Python 기반의 심플한 웹 애플리케이션
- **작성일**: 2026년 3월 11일

## 🎯 핵심 기능 요구사항

### 1. 기본 기능
- **DB 스키마 정보 입력**: 테이블 구조, 인덱스, 관계 정보 등
- **원본 쿼리 입력**: 튜닝이 필요한 SQL 쿼리 입력
- **AI 튜닝 요청**: H-Chat API를 통한 쿼리 최적화 수행
- **결과 비교 화면**: 원본 vs 튜닝된 쿼리 비교 표시

### 2. 화면 구성 (예상)

## 🎯 **확정된 UI/UX 설계**

### 📱 **단계별 화면 구성** ✅
1. **Step 1: DB 스키마 입력 페이지**
   - 여러 테이블 스키마 입력 (JOIN 관계 고려)
   - "다음 단계" 버튼으로 이동
   
2. **Step 2: 쿼리 입력 페이지**
   - 원본 쿼리 입력
   - 스키마 정보 요약 표시 (참고용)
   - "튜닝 실행" 버튼
   
3. **Step 3: 결과 비교 페이지**
   - 좌측: 원본 쿼리 (Before)
   - 우측: 튜닝된 쿼리 (After)
   - 변경된 부분 하이라이트 표시
   - 수정 근거를 주석 형태로 표시

### 🏗️ **스키마 입력 구조** ✅
```
📋 간단한 텍스트 입력 방식 (다중 테이블 지원)
┌─────────────────────────────────────┐
│ 테이블 스키마 정보 입력 (DDL 형식)     │
│ ----------------------------------- │
│ -- 테이블 1                         │
│ CREATE TABLE users (                │
│   id NUMBER PRIMARY KEY,            │
│   name VARCHAR2(100),               │
│   dept_id NUMBER                    │
│ );                                  │
│                                     │
│ -- 테이블 2                         │
│ CREATE TABLE departments (          │
│   id NUMBER PRIMARY KEY,            │
│   dept_name VARCHAR2(50)            │
│ );                                  │
│                                     │
│ -- 인덱스 정보                      │
│ CREATE INDEX idx_users_dept         │
│   ON users(dept_id);                │
│                                     │
│ -- 테이블 통계 (선택사항)           │
│ -- users: 약 100만건                │
│ -- departments: 약 50건             │
└─────────────────────────────────────┘
```

### 🎨 **결과 화면 레이아웃** ✅
```
┌─────────────────────┬─────────────────────┐
│    원본 쿼리 (Before)  │   튜닝된 쿼리 (After) │
├─────────────────────┼─────────────────────┤
│ SELECT *            │ SELECT u.id, u.name │ ← 변경된 부분 하이라이트
│ FROM users u        │ FROM users u        │
│ JOIN departments d  │ JOIN departments d  │
│   ON u.dept_id = d.id│   ON u.dept_id = d.id│
│ WHERE u.name LIKE   │ WHERE u.name LIKE   │
│   '%김%'            │   '%김%'            │ 
│                     │ -- 성능 개선: SELECT *│ ← 주석으로 설명
│                     │ -- 대신 필요한 컬럼만  │
│                     │ -- 명시하여 I/O 최적화 │
└─────────────────────┴─────────────────────┘
```

### 2. DB 스키마 입력 방식 (Tibero 전용)

**필수 입력 항목들:**

#### 📋 테이블 정보
- [ ] **테이블명**: 쿼리에서 사용되는 모든 테이블명
- [ ] **컬럼 정보**:

  - 기본값, 제약조건 정보
- [ ] **기본키(PK)**: PRIMARY KEY 컬럼 정보
- [ ] **외래키(FK)**: FOREIGN KEY 관계 정보

#### 🔍 인덱스 정보  
- [ ] **인덱스명**: 기존 생성된 인덱스 목록
- [ ] **인덱스 구성**: 어떤 컬럼들로 구성되어 있는지

- [ ] **인덱스 통계**: 선택도, 카디널리티 정보 (가능한 경우)

#### 📊 테이블 통계 정보
- [ ] **데이터 건수**: 각 테이블의 대략적인 row count
- [ ] **데이터 크기**: 테이블별 용량 정보 (옵션)
- [ ] **업데이트 빈도**: 자주 변경되는 테이블인지 (옵션)

#### 🔗 관계 정보
- [ ] **테이블 간 관계**: JOIN 관계 및 카디널리티
- [ ] **참조 무결성**: FK 제약조건 정보
